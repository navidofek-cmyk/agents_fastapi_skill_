# AGENTS

Project rules:
- Keep changes minimal.
- Keep business logic in the service layer.
- Testing is required for behavior changes.
- The `tester` role is mandatory for every code change before a task is considered complete.

Subagents:
- planner: planning only.
- coder: implementation.
- tester: required tests and verification for every code change.
- reviewer: review for bugs and gaps.

See [docs/agent_matrix.md](/home/ivand/projects/rd/vibe_coding_course/lesson_02/agents_fastapi_skill_/docs/agent_matrix.md) for the detailed execution matrix, role boundaries, and done criteria.
